Void Linux is a general purpose operating system, based on the monolithic Linux kernel. It has a stable release model. It supports both musl and GNU libc implementation letting the developers use whatever best fits them.

Pros: Void uses the runit system which in most cases is faster than the older styled systemd init. Void is minimalist, it allows the user to do whatever the hell they want, and unlike void, it won’t collapse into a black hole if you miss a package update, on that note, the Void’s XBPS package manager is blazing fast compared to basically every other package manager

Cons Since void linux is minimalist and forces you to basically build your own system from scratch, its not for beginners, and musl breaks some packages that depend on glibc

Software management: Void uses the XPBS package manager

Availability: ISO are available in x86_64, i636 and arm architectures
